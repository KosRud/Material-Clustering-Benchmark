Default value of Parameters.p is 2.5, which we experimentally confirmed to be optimal for our dataset (WorkGeneration.KHMp). For values above p=3.0 we saw a sharp decline in quality for one of the video files.

Integer powers are optimized by the compiler to use multiplication instead, which is faster. For non-integer powers pow(a,b) will be replaced with exp2(b * log2(a)), because there is no hardware pow().

According to [1]: p=3.5 is the optimal value for 2-dimensional data in a general case; p=3 produces almost as good clustering quality as p=3.5; any value above p=2 should generally outperform K-means; for higher dimensions values of p snould be higher.

In our application the samples are located on the edges of a triangle in two-dimensional space, which could explain lower value of p being optimal.

[1] Zhang, B., 2000. Generalized k-harmonic means–boosting in unsupervised learning. HP LABORATORIES TECHNICAL REPORT HPL, 137.
